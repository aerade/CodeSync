Download section on website homepage
What & Why
Добавить на главную страницу сайта CodeSync аккуратный раздел «Скачать десктоп-приложение», чтобы любой посетитель (например, друг по ссылке на опубликованный сайт) мог в один клик скачать готовый установщик для своей ОС — обычный графический инсталлятор (.dmg / .exe / .AppImage), без терминала и PowerShell. Раздел должен выглядеть в той же стилистике, что и остальной лендинг (тёмный фон, glassmorphism, тонкие границы, акцентные цвета).

Done looks like
На главной странице сайта (artifacts/codesync, путь /) появляется новый раздел с заголовком вроде «Десктоп-приложение» / «Установить на свой компьютер», визуально вписанный между существующими секциями (логично — между «Features» и финальным CTA).
В разделе автоматически определяется ОС посетителя (macOS / Windows / Linux) и крупная primary-кнопка «Скачать для <ОС>» сразу ведёт на правильный установщик последнего релиза.
Под главной кнопкой компактно показаны ссылки на установщики для остальных платформ (например, «macOS · Windows · Linux» с указанием расширения файла и размера, если доступно).
При клике по кнопке у пользователя начинается прямое скачивание файла установщика (.dmg, .exe или .AppImage) — без перехода в терминал, без копирования команд, без открытия GitHub-страницы.
Информация о последнем релизе (версия, дата, ссылки на ассеты) подтягивается из GitHub Releases API репозитория aerade/CodeSync — то есть кнопка автоматически начинает отдавать новый билд, как только в CI публикуется новый тег.
Если релизов ещё нет, раздел показывает аккуратное состояние «Скоро будет доступно» с пояснением и ссылкой на репозиторий — без сломанных кнопок и без ошибок в консоли.
Если запрос к GitHub API упал (rate limit, сеть), раздел показывает понятный фолбэк и не ломает остальной лендинг.
Раздел адаптивен: на мобильных кнопка и список платформ перестраиваются в одну колонку и остаются читаемыми.
Визуально раздел соответствует существующему стилю: тёмный фон #030303, белый текст с прозрачностями, тонкие рамки rgba(255,255,255,0.08), glassmorphism, мягкие радиальные подсветки, мини-иконки/бейджи в духе акцентных цветов лендинга (синий/зелёный/оранжевый).
Out of scope
Сборка и публикация самих установщиков в CI — это отдельная работа (релизный пайплайн уже существует / планируется отдельно). Здесь мы только потребляем то, что уже опубликовано в GitHub Releases.
Подписание установщиков, SmartScreen, нотаризация macOS — отдельные задачи.
Изменения в самом десктоп-приложении (artifacts/desktop) и API-сервере (artifacts/api-server) — не нужны.
Аналитика/трекинг кликов по кнопке скачивания.
Авто-обновление и release notes на сайте — здесь только кнопка скачать + версия/дата.
Steps
Получение данных о последнем релизе. Сделать небольшой клиентский хук, который при загрузке главной страницы запрашивает последний релиз с GitHub Releases API для репозитория aerade/CodeSync, разбирает список ассетов (.dmg, .exe, .AppImage) и сопоставляет их с платформами (macOS / Windows / Linux), извлекая имя файла, размер и URL. Корректно обрабатывать состояния «загружается / нет релиза / ошибка».

Определение ОС посетителя. Добавить простой детектор платформы по navigator.userAgent / navigator.platform (с учётом современного userAgentData, если доступен). На основе этого выбирать, какой установщик показывать как основной.

Сборка визуального раздела. Сделать новый компонент download-секции в стилистике существующего лендинга: заголовок, короткое описание, крупная primary-кнопка «Скачать для <ОС>» (с иконкой платформы), под ней — компактный ряд ссылок для остальных платформ с расширением файла и размером, плюс мелким шрифтом — текущая версия и дата релиза. Анимации и наведения — в духе уже используемых на странице (framer-motion, плавные hover-эффекты).

Состояния «нет релиза» и «ошибка». Если последний релиз не найден — показать понятный плейсхолдер «Десктоп-приложение скоро будет доступно для скачивания» с второстепенной ссылкой на репозиторий. Если запрос к GitHub упал — показать аналогичный фолбэк, не ломая страницу.

Встраивание в главную страницу. Подключить новый раздел в home.tsx между блоком «Features» и финальным CTA, добавить ссылку на этот якорь в верхней навигации (например, «Скачать»), чтобы кнопка в шапке плавно скроллила к разделу.

Проверка. Локально открыть сайт, убедиться, что: раздел отображается в нужном месте; основная кнопка соответствует ОС, с которой смотрят; клик действительно инициирует скачивание файла (а не уводит в терминал/командную строку); фолбэки отрабатывают, если релизов нет; раздел нормально выглядит на мобильной ширине.
